=== Versão 1.52

.Validação de recursos
Para garantir a confiabilidade, Data Classification agora valida a memória e os recursos de disco disponíveis durante a verificação, geração de relatórios e ações. Quando o uso de memória excede 95% ou o espaço de disco cai abaixo de 6 GB, Data Classification pausa automaticamente as operações. As operações são retomadas assim que os recursos forem suficientes.

Para monitorar os recursos do seu sistema, veja link:https://docs.netapp.com/us-en/data-services-data-classification/task-health-monitor.html["Monitore a integridade da Classificação de Dados."^].

.Aumento do limite para copiar e mover
As funções de copiar e mover agora são compatíveis para destinos CIFS. O limite de tamanho por arquivo para ambas as ações aumentou para 1 GB em relação ao limite anterior de 50 MB. Os limites de operação para ambas as ações aumentaram para 1 TB ou 5 milhões de arquivos.

Para mais informações, consulte link:https://docs.netapp.com/us-en/data-services-data-classification/task-copy-move.html["Copiar, mover e excluir dados"^].

.Permissões de configuração de integração do Active Directory
As alterações nas configurações de integração do Active Directory (AD) na Data Classification agora estão restritas a administradores. Visualizadores não têm mais a capacidade de modificar as configurações do AD.

Para mais informações sobre a função Classification viewer, consulte link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html#data-service-roles["Funções do serviço de dados"^].

.Suporte para RHEL 9.7
A Data Classification adicionou o Red Hat Enterprise Linux v9.7 para instalação manual local, incluindo implantações em ambientes de segurança restrita.

Os seguintes sistemas operacionais requerem o uso do mecanismo de contêiner Podman e exigem Data Classification versão 1.30 ou superior: Red Hat Enterprise Linux versão 8.8, 8.10, 9.0, 9.1, 9.2, 9.3, 9.4, 9.5, 9.6 e 9.7.

.Suporte a proxy transparente
A Classificação de Dados agora oferece suporte a configurações de proxy transparente, permitindo uma implantação mais tranquila em ambientes de rede com proxy ou restrições.

Para obter mais informações sobre a implantação em ambientes restritos, consulte link:https://docs.netapp.com/us-en/data-services-data-classification/task-deploy-compliance-dark-site.html["Instalar o NetApp Data Classification em um host Linux sem acesso à Internet"^].
