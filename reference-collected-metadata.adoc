= Metadados coletados de fontes de dados na NetApp Data Classification
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Data Classification coleta metadados durante a análise de suas fontes de dados e sistemas. A maioria dos metadados está disponível, mas alguns atributos não estão disponíveis para todos os protocolos.

[cols="15,25,25,25"]
|===
|  | *Metadados* | *CIFS* | *NFS* 


.3+| *Carimbos de tempo* | _Tempo de criação_ | Disponível | Não disponível (sem suporte no Linux) 


| _Último horário de acesso_ | Disponível | Disponível 


| _Última modificação_ | Disponível | Disponível 


.2+| *Permissões* | _Permissões abertas_ | Se o grupo "TODOS" tiver acesso ao arquivo, ele será considerado "Aberto à organização". | Se "Outros" tiver acesso ao arquivo, ele será considerado "Aberto à organização". 


| _Acesso de usuários/grupos_ | As informações de usuários e grupos são obtidas do LDAP | Não disponível (os usuários do NFS geralmente são gerenciados localmente no servidor, portanto, o mesmo indivíduo pode ter um UID diferente em cada servidor) 
|===
[NOTE]
====
* A Classificação de Dados não extrai o "último horário de acesso" das fontes de dados do banco de dados.
* Versões mais antigas do sistema operacional Windows (por exemplo, Windows 7 e Windows 8) desativam por padrão a coleta do atributo "último acesso", pois isso pode afetar o desempenho do sistema. Quando esse atributo não é coletado, as análises de Classificação de Dados baseadas no "último acesso" serão afetadas. Você pode habilitar a coleta do "último acesso" nesses sistemas Windows mais antigos, se necessário.


====


[[last-accessed-time]]
== Último horário de acesso

Quando o Data Classification extrai dados de compartilhamentos de arquivos, o sistema operacional considera isso um evento de acesso e atualiza o "último horário de acesso" de acordo. Após a verificação, o Data Classification tenta reverter o último horário de acesso para o registro de data e hora original. Se o Data Classification não tiver permissões de gravação de atributos no CIFS ou permissões de gravação no NFS, o sistema não poderá reverter o último horário de acesso para o registro de data e hora original.

Por padrão, Data Classification não analisa arquivos quando não possui permissões de gravação para restaurar o carimbo de data/hora original. Se seus dados puderem tolerar que o horário do último acesso não seja redefinido para o carimbo de data/hora original, habilite a opção *Analisar sem permissões de gravação* na página de configuração do sistema.

image:screenshot_scan_missing_permissions.png["Uma captura de tela mostrando a opção que você pode usar para que o Data Classification analise seus arquivos mesmo que não tenha as permissões corretas para reverter o último acesso ao registro de data e hora original."]

Essa funcionalidade é aplicável a sistemas ONTAP locais, Cloud Volumes ONTAP, Azure NetApp Files, Amazon FSx for NetApp ONTAP e compartilhamentos de arquivos de terceiros.


NOTE: Os volumes ONTAP configurados com SnapLock têm permissões somente de leitura e também não podem reverter o último acesso para o carimbo de data/hora original.
